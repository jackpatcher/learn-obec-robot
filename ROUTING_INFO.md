# การใช้งาน Routing ในแอปพลิเคชัน

## ภาพรวม
แอปพลิเคชันได้ถูกอัพเดทให้รองรับ Vue Router แล้ว ตอนนี้แต่ละแท็บจะมี URL ของตัวเอง

## URL Routes ที่มีให้ใช้งาน

### หน้าหลัก
- `/` หรือ `/#/home` - หน้าแรก
- `/#/random` - สุ่มตัวเลข
- `/#/timer` - จับเวลา
- `/#/countdown` - นับถอยหลัง

### ระดับกลาง
- `/#/mid/rules` - กติการะดับกลาง
- `/#/mid/field` - สร้างสนามระดับกลาง
- `/#/mid/timer` - จับเวลา 3 นาที (ระดับกลาง)
- `/#/mid/teams` - ตั้งค่าทีมระดับกลาง
- `/#/mid/results` - ผลการแข่งขันระดับกลาง

### ระดับสูง
- `/#/high/rules` - กติการะดับสูง
- `/#/high/field` - สร้างสนามระดับสูง
- `/#/high/timer` - จับเวลา 5 นาที (ระดับสูง)
- `/#/high/teams` - ตั้งค่าทีมระดับสูง
- `/#/high/results` - ผลการแข่งขันระดับสูง

## คุณสมบัติ

1. **Browser History**: สามารถใช้ปุ่ม Back/Forward ของเบราว์เซอร์เพื่อนำทางได้
2. **Bookmark**: สามารถบุ๊คมาร์ค URL ของแต่ละหน้าได้
3. **Direct Link**: สามารถแชร์ลิงก์ไปยังหน้าใดหน้าหนึ่งโดยตรงได้
4. **Auto Expand**: เมื่อเข้าหน้าระดับกลางหรือระดับสูง เมนูจะขยายอัตโนมัติ

## ตัวอย่างการใช้งาน

### แชร์ลิงก์ไปยังหน้าจับเวลาระดับกลาง
```
file:///path/to/midhigh.html#/mid/timer
```
หรือถ้าอัพโหลดขึ้น server
```
https://yourdomain.com/midhigh.html#/mid/timer
```

### เปิดหน้าสร้างสนามระดับสูงโดยตรง
```
file:///path/to/midhigh.html#/high/field
```

## การทำงาน

- ใช้ Vue Router ในโหมด hash (#) เพื่อรองรับการทำงานแบบ static file
- URL จะอัพเดทอัตโนมัติเมื่อคลิกเมนู
- หน้าจะเปลี่ยนอัตโนมัติเมื่อ URL เปลี่ยน
- รองรับการนำทางด้วยปุ่ม Back/Forward ของเบราว์เซอร์

## หมายเหตุ

- Router ใช้โหมด `hash` (มี # ใน URL) เพื่อให้ทำงานได้กับไฟล์ HTML แบบ static
- หากต้องการใช้ `history mode` (ไม่มี #) จะต้องตั้งค่า server ให้รองรับ
